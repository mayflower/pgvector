## Recommended Defaults

- Use `SIMD_BUILD=portable` for package builds.
- Use `SIMD_BUILD=native` only for local benchmark binaries.
- Keep scalar force GUCs available for parity and incident fallback.

## Validation Scope

- Host: Apple M4 arm64, PostgreSQL 16.13, portable build restored after native build validation.
- Coverage: SIMD profile wrapper, dense-only, BM25, hybrid, no-lexical-match, and delta-heavy smoke cases.
- Parity: every smoke case passed exact top-k parity or accepted set-level tie parity.
- Caveat: this 200-row smoke profile is intended to catch dispatch, GUC, and parity regressions. Fixed SQL/index overhead dominates these timings, so speedups here are not representative of production-size datasets.
- Missing local hardware: amd64 AVX2/AVX-512 validation was not available on this host. Run the same matrix on an amd64 machine before enabling `hnsw.tq_graph_avx512_weighted=auto` by default.

## Merge Gate

- `SIMD_BUILD=none`: compile passed earlier in this validation pass.
- `SIMD_BUILD=portable`: compile, install, and `turbohybrid_simd` regression pass.
- `SIMD_BUILD=native`: compile pass.
- `bench-weighted-tq`: `hnsw.tq_graph_avx512_weighted=off` and `on` smoke runs passed on arm64 without selecting AVX-512.
